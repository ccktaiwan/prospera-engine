## D-4. State machine drift — runtime vs specification

**Status:** Open
**Priority:** P1
**Type:** Functional rewrite (not cleanup)
**Origin:** Audit comparison of three state machine declarations in this repo

### Problem

Three locations in this repository define a state machine, and they
disagree with one another:

| Source | States | Events |
|---|---|---|
| `engine_runtime.py` (runtime, repo root) | INIT, RUNNING, HALTED | BOOT, STOP |
| `02_SPECIFICATIONS/STATE_MODEL.md` (canonical spec) | INITIALIZED, IDLE, PROCESSING, COMMITTED, FAILED | (event-driven) |
| `tests/test_state_model.py` (test) | matches STATE_MODEL.md | — |

`02_kernel/governance/authority_matrix.py` previously held a fourth variant
(INIT/ACTIVE/SUSPENDED/COMPLETED/FAILED/TERMINATED/ARCHIVED); that file
was removed in `8e7d7aa` as a structural violation, but its presence
confirms the inconsistency had spread beyond two locations.

### Why this is a P1 issue

`03_GOVERNANCE/ARCHITECTURE_CONSTRAINTS.md` §3 explicitly requires:

> "Runtime code must follow both architecture and specification constraints."

> "Any code change that contradicts architecture documentation must be rejected."

`engine_runtime.py` directly contradicts `02_SPECIFICATIONS/STATE_MODEL.md`.
This is a Codex-level non-compliance, not a stylistic concern.

### Why deferred

Resolving this is not a cleanup operation. It requires:

1. Deciding which model is authoritative (the spec is the obvious
   canonical answer, but the simpler runtime model may have downstream
   consumers).
2. Rewriting `engine_runtime.py` to implement the 5-state FSM from
   STATE_MODEL.md, including the event lifecycle described in
   EVENT_SCHEMA.md §6.
3. Updating `tests/manual/test_engine.py` (currently exercises
   BOOT/STOP) to match the new transitions.
4. Adding proper pytest coverage in `tests/` (the rewritten
   `test_state_model.py` validates the spec data only — there is no
   test that runs the actual engine through the spec's transitions).

### Suggested resolution path

1. Open a P1 issue in `ccktaiwan/prospera-engine` titled
   "P1: engine_runtime.py state machine does not match STATE_MODEL.md spec"
2. Author a small ADR in `01_ARCHITECTURE/` declaring spec as canonical
   (formalises the §3 rule for this specific case)
3. Rewrite `engine_runtime.py` per STATE_MODEL.md
4. Update the smoke test in `tests/manual/test_engine.py`
5. Add a pytest exercising the actual engine through canonical transitions
